# 开发日志系统使用指南

## 概述

开发日志系统用于记录项目的功能开发、优化和维护历史，以时间线的形式展示在网站的"开发日志"页面。

## 功能特性

- ✅ 时间线可视化展示
- ✅ 按类别筛选（里程碑、新功能、优化、重构、Bug 修复、维护）
- ✅ 按月份分组
- ✅ 支持标签和详细信息
- ✅ 影响级别标识（高/中/低）
- ✅ 响应式设计

## 数据存储

开发日志数据存储在 `site-content.json` 文件的 `devLog` 数组中。

### 数据结构

```json
{
  "id": "devlog-001",
  "date": "2026-04-28",
  "title": "功能标题",
  "category": "feature",
  "description": "简短描述",
  "tags": ["标签1", "标签2"],
  "details": [
    "详细信息第一条",
    "详细信息第二条"
  ],
  "impact": "high"
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | 是 | 唯一标识符，格式：`devlog-{timestamp}` |
| `date` | string | 是 | 日期，格式：`YYYY-MM-DD` |
| `title` | string | 是 | 标题 |
| `category` | string | 是 | 类别：`milestone`/`feature`/`enhancement`/`refactor`/`bugfix`/`maintenance` |
| `description` | string | 是 | 简短描述 |
| `tags` | array | 否 | 标签数组 |
| `details` | array | 否 | 详细信息数组 |
| `impact` | string | 否 | 影响级别：`low`/`medium`/`high`，默认 `medium` |

### 类别说明

- **milestone** (里程碑): 重大版本发布、重要功能完成
- **feature** (新功能): 新增功能特性
- **enhancement** (优化): 性能优化、体验改进
- **refactor** (重构): 代码重构、架构调整
- **bugfix** (Bug 修复): 问题修复
- **maintenance** (维护): 依赖更新、文档维护

## 添加开发日志

### 方法一：使用自动化脚本（推荐）

```bash
node add-devlog.js
```

脚本会引导你输入以下信息：
1. 标题
2. 类别（1-6 选择）
3. 简短描述
4. 标签（逗号分隔）
5. 详细信息（每行一条，空行结束）
6. 影响级别（1-3 选择）
7. 日期（默认今天）

完成后会自动更新 `site-content.json` 文件。

### 方法二：手动编辑

1. 打开 `site-content.json`
2. 在 `devLog` 数组的开头添加新记录
3. 确保 JSON 格式正确
4. 保存文件

### 示例

```bash
# 使用脚本添加
$ node add-devlog.js

=== 创建新的开发日志 ===

标题: 添加用户认证功能
选择类别:
  1. 里程碑
  2. 新功能
  3. 优化
  4. 重构
  5. Bug 修复
  6. 维护
类别 (1-6): 2
简短描述: 实现用户登录、注册和权限管理
标签 (用逗号分隔): 认证, 安全, 用户管理
详细信息 (每行一条，输入空行结束):
  - 实现 JWT 认证机制
  - 添加用户注册和登录页面
  - 实现权限中间件
  - 
影响级别:
  1. Low (低)
  2. Medium (中)
  3. High (高)
影响级别 (1-3, 默认 2): 3
日期 (YYYY-MM-DD, 默认 2026-04-28): 

✅ 开发日志已添加！
```

## 提交更改

添加开发日志后，记得提交到 Git：

```bash
git add site-content.json
git commit -m "docs: 添加开发日志 - [功能标题]"
git push origin main
```

推送后，Vercel 会自动部署更新。

## 最佳实践

1. **及时记录**: 每次完成功能开发或重要修改后立即添加日志
2. **描述清晰**: 标题简洁明了，描述准确具体
3. **合理分类**: 选择最合适的类别
4. **添加标签**: 使用标签方便后续检索
5. **详细信息**: 对于重要功能，添加详细的实现要点
6. **影响评估**: 准确评估功能的影响级别

## 页面展示

开发日志会在网站的"开发日志"页面展示：

- 按月份分组显示
- 时间线可视化
- 支持按类别筛选
- 悬停显示详细信息
- 响应式布局

## 故障排查

### 日志不显示

1. 检查 `site-content.json` 格式是否正确
2. 确认 `devLog` 数组存在且有数据
3. 检查浏览器控制台是否有错误
4. 清除浏览器缓存

### 筛选不工作

1. 确认 `category` 字段值正确
2. 检查 JavaScript 是否正常加载
3. 查看浏览器控制台错误信息

## 技术实现

- **前端**: 原生 JavaScript

- **动画**: CSS transitions + animations
